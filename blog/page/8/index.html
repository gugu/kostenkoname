Костыльных дел мастер

Jul 6th, 2010

Постоянно путаю svn, hg, git. В репозитории svn пишу hg up, git commit. Чтобы разобраться как-то с зоопарком систем контроля версия сделал команду vcs:

if [ -e .svn ]; then
    svn $*;
elif [-e .git]; then
    git $*
elif [ -e .hg ]; then
    hg $*
fi

как бонус можно и так:

elif [-e CVS ]; then
  echo "Еееебать динозавр"
fi

Upload Preview

Jul 5th, 2010

Случайно наткнулся на такую бесполезную, на прикольную штуку, которая работает в IE и Firefox, а скоро должна появиться в остальных браузерах (ибо W3C).

Допустим, Вам нужно сделать загрузку картинки и показать её preview на странице. Обычно это делается через невидимые ифреймы и другую хуету.

Но можно показать preview картинки даже не загружая её на сервер!

Как это можно сделать в IE:

<img id="i"></img>
<input onchange="document.getElementById('i').src=this.value" type="file" id="f"></input>

На самом деле это просто дырка в безопасности. Полезная. А вот Firefox поступает гораздо умнее – он поддерживает W3C File API. У поля выбора файлов появляется аттрибут files:

var f = document.getElementById('f')
text_content = f.files[0].getAsText() //текстовое представление
document.getElementById('i').src = f.files[0].getAsDataURL()

Опера показала заботу о безопасности (скрыла путь к файлу) и безразличие к W3C File API.

Chromium 6 показал частичную поддержку File API (можно посмотреть размер и mime-тип файла), но нельзя получить его содержимое.

На данный момент эта фича бесполезна, но, мне кажется, скоро остальные браузеры подтянутся (не считая IE, конечно)

Можно, конечно же, попробовать

Как сделать, чтобы сообщения из вашего гавнобложика постились сразу в Твиттер?

Всё просто - подключаем FeedBurner, после чего Publicize => Socialize и подключаем твиттер.
